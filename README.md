# Generate Script Manual

Репозиторий для скриптов ручной выгрузки данных и сопутствующей документации.

## Структура проекта

- **Сводка по сети и порядку вызовов:** [Docs/Справочник_скрипты_HTTP_запросы_и_последовательность.md](Docs/Справочник_скрипты_HTTP_запросы_и_последовательность.md) — все скрипты из `Script/`: методы, пути, **payload**, последовательность шагов.
- **Script/** — скрипты для запуска через консоль разработчика на странице стенда или приложения.
  - Загрузка профилей героев: **`Profile_GP_LOAD_file.js`** — панель ~**920px**: выбор **стенда** (`PROM/PSI`) и **контура** (`ALPHA/SIGMA`), блок параметров (задержки и retry **в одну строку**, чекбоксы фото **в ряд**, подсказки `title`); **журнал работы** на панели (подробный ход сбора; в консоли — кратко о старте панели и итоге прогона); три кнопки запуска **в один ряд**; ТН из поля, из `.txt` или из `TAB_NUMS`; фото через Blob и ссылки «Скачать» при блокировке автоскачивания. См. [Docs/Скрипт_загрузка_профиля_герои.md](Docs/Скрипт_загрузка_профиля_герои.md).
  - Выгрузка файлов gamification: **`File_DB_Load_GP.js`** — **IIFE** (повторная вставка в консоль без перезагрузки), POST `file-download`, панель (~960px), выбор **стенда** (`PROM/PSI`) и **контура** (`ALPHA/SIGMA`), **«Отметить всё» / «Снять»** справа, сводка **(отмечено: N)**, дата наград в блоке пауз, основные **три кнопки**, «Рейтинг» / «Заказы», **журнал работы** повыше по высоте, сброс состояния при **«Закрыть»**; при HTTP 200 и JSON с `success: false` файл не сохраняется.
  - Адресная книга: **`AddressBook_export.js`** — панель до **700px** по ширине, **IIFE** (повторная вставка в консоль без конфликта `const`); отдельный стенд **ALPHA** (без выбора контура/стенда), один общий ввод, режим разбора (`табельный` с нормализацией / `значения для поиска` без нормализации), **3 кнопки** по полю и **3 кнопки** по файлу (`Search → empInfoFull`, `Только Search`, `Только empInfoFull`); компактный блок пауз; запросы с приоритетом на **origin вкладки** (fallback `ADDRESSBOOK_ORIGINS.ALPHA`), при нескольких `employeeId` в ответе search — **empInfoFull** по каждому; см. [Docs/Скрипт_AddressBook_export.md](Docs/Скрипт_AddressBook_export.md) (v1.16).
  - Параметры: **`Parameters_Actual_Export.js`** — **три вкладки**: выгрузка (`[Выгрузка]`), создание (**`param-create`**, ⬇ на вкладке 2 — ACTUAL + детализация **`parameterTypes`**, перед созданием — условная подгрузка кэшей), правка (**`param-update`**, на вкладке 3 — отдельная ⬇ для справочников, поиск `parameterCode` по части текста, автоподстановка `objectId/type/status/version`, сверка связки `objectId↔parameterCode`); **стенд/контур**; **`PARAMETER_TYPE_OPTIONS`**. См. [Docs/Скрипт_выгрузка_актуальных_параметров_Parameters_Actual_Export.md](Docs/Скрипт_выгрузка_актуальных_параметров_Parameters_Actual_Export.md) (**v3.2**).
  - Турниры: **`Tournament_LeadersForAdmin.js`** — GET `leadersForAdmin`, панель **IIFE**; выбор **стенда** (`PROM/PSI`) и **контура** (`ALPHA/SIGMA`), префикс имени файла, пауза между запросами; поле кодов с **`DEFAULT_TOURNAMENT_CODES_TEXTAREA`**; **журнал работы** (лента); в JSON — успешные ответы; при ошибке — **`body.tournament.error`** внутри записи с `success: true`, `status: "ERROR"` (одна строка при плоском экспорте); **«0 участников»** (`buildLeadersExportRecordArray`); без скачивания пустого `{}`; CSV — общий верх колонок (`<select>` + «Другой…»), фильтр статусов **свой у каждой кнопки** SHEDULE/LIST. Док.: [Docs/Скрипт_турниры_leadersForAdmin.md](Docs/Скрипт_турниры_leadersForAdmin.md) (актуально **v1.23**).
- **Docs/** — подробная документация по отдельным скриптам; общий справочник HTTP — [Docs/Справочник_скрипты_HTTP_запросы_и_последовательность.md](Docs/Справочник_скрипты_HTTP_запросы_и_последовательность.md).
- **log/** — каталог для логов (при наличии логирования в проекте).
- **POST/** — каталог **в `.gitignore`**, в репозиторий **не коммитится**. После клонирования или обновления выполните `./post_txt_sync.sh` — в `POST/` появятся копии с суффиксом **`.txt`**: каждый `Script/*.js`, каждый `Docs/*.md`, `README.md`, а также **`.gitignore`** и **`post_txt_sync.sh`** из корня (имена вида `.gitignore.txt`, `post_txt_sync.sh.txt`). Удобно для вставки в системы, где файл должен иметь расширение `.txt`. Команда: `chmod +x post_txt_sync.sh && ./post_txt_sync.sh` из корня репозитория.
- **post_txt_sync.sh** — скрипт пересборки каталога `POST/` (файл в репозитории; сам `POST/` — нет).

## Скрипт выгрузки файлов gamification (`File_DB_Load_GP.js`)

**Задача:** инициировать скачивание выгрузок (CSV и др.) с API gamification в браузере пользователя, используя сессию текущей вкладки (`credentials: "include"`), без вшивания куков и табельных номеров.

**Решение:** фиксированный список задач (основные три запроса + 10 вариантов рейтинга + 10 вариантов заказов); для каждой задачи — `fetch` POST с JSON-телом; ответ сохраняется как файл. Пакет — **последовательно** или с **перекрытием** (чекбокс на панели), паузы настраиваются в блоке **«Пакет: паузы»**, дата `dateFrom` для наград — полем **«Награды с»**. Код обёрнут в **IIFE**: повторная вставка **всего файла** в консоль **без перезагрузки вкладки**; панель `id=fileDlGamificationPanelRoot`, при «Закрыть» вызывается **`fileDlDetachPanelAndResetRuntime`**. Панель (~960px): сводка с **(отмечено: N)**; **«Отметить всё» / «Снять отметки»** справа от выбора стенда; основные кнопки в три колонки (в т.ч. **«Награды: (LIST REWARD)»**, **«Посещения»**); «Рейтинг» / «Заказы»; **журнал работы**; «Скачать выделенное»; «Закрыть». Функции **`downloadAllJobs()`** и остальные доступны **только внутри IIFE**, не из глобальной консоли.

**Документация:** полное ТЗ по константам, задачам, функциям, UI и логам — в файле  
[Docs/Скрипт_выгрузка_файлов_gamification_File_DB_Load_GP.md](Docs/Скрипт_выгрузка_файлов_gamification_File_DB_Load_GP.md) (актуальная версия документа — см. раздел «История версий» внизу того файла; сейчас **v1.17**).

**Краткий перечень сущностей в коде (внутри IIFE):**

| Имя | Назначение |
|-----|------------|
| `DEFAULT_FILE_DL_STAND`, `DEFAULT_FILE_DL_CONTOUR`, `FILE_DL_ACTIVE_STAND`, `FILE_DL_ACTIVE_CONTOUR`, `STAND_ORIGINS` | Пара стенд/контур по умолчанию, выбор на панели, таблица origin |
| `DOWNLOAD_JOBS` | Три основные выгрузки: турниры, **«Награды: (LIST REWARD)»**, **«Посещения»** |
| `RATING_GROUP_JOBS` | 10 POST на рейтинг (`businessBlock` + `timePeriod`) |
| `ORDERS_GROUP_JOBS` | 10 POST на заказы (`businessBlock` + `listType`, в т.ч. `NONSEASON`) |
| `getAllDownloadJobs`, `downloadOneJob`, `downloadJobsSequentially`, `downloadJobsStaggered`, `downloadJobsBatch` | Сборка списка, один запрос, пакет последовательно или с перекрытием |
| `downloadCheckedPanelJobs`, `downloadRatingGroupOnly`, `downloadOrdersGroupOnly` | Пакеты с панели (режим по `FILE_DL_USE_STAGGER`) |
| `fileDlDetachPanelAndResetRuntime` | Сброс ссылок на DOM и рабочих переменных при «Закрыть» и перед повторным открытием панели |
| `downloadAllJobs` | Все задачи подряд (только внутри скрипта; не вызывается из глобальной консоли) |
| `startDownloadPanel` | Удаляет прежнюю панель при наличии, отрисовывает UI (автовызов в конце IIFE) |

**Запуск:** открыть нужный стенд в браузере → DevTools → Console → вставить **полное** содержимое `Script/File_DB_Load_GP.js` → Enter. Повторный запуск — снова вставить весь файл (или закрыть панель и вставить снова).

## Скрипт загрузки профиля героев

**Задача:** пакетно запросить профили по табельным номерам через API геймификации и сохранить ответы в JSON на диск пользователя.

**Решение (основной файл `Profile_GP_LOAD_file.js`):** IIFE, панель без перезагрузки страницы; POST с куками для SIGMA; опционально вырезание base64 фото из JSON (`photoDataInfo`) и сохранение снимков в файлы; поиск контейнера `photoData` / `photoDataKpk` во вложенной структуре `body`; декодирование base64 с padding и URL-safe; ручные ссылки на панели, если браузер блокирует программное скачивание после `fetch`. На панели: компактный блок параметров, **журнал работы** (`profileGpPanelEcho`: подробности на панели; в консоли — краткий старт и итог), сброс привязки журнала при «Закрыть» и при открытии новой панели.

**Запуск:** страница стенда → консоль → вставить `Script/Profile_GP_LOAD_file.js`.

Полный перечень констант, функций и сценариев — в [Docs/Скрипт_загрузка_профиля_герои.md](Docs/Скрипт_загрузка_профиля_герои.md). В документации не приводятся полные URL и реальные табельные номера; используются маски и примеры вида `00001234`.
