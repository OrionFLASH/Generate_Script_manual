# Generate Script Manual

Репозиторий для скриптов ручной выгрузки данных и сопутствующей документации.

## Структура проекта

- **Script/** — скрипты для запуска через консоль разработчика на странице стенда или приложения.
  - Загрузка профилей героев: **`Profile_GP_LOAD_file.js`** — панель (стенд ALPHA/SIGMA, задержки, retry, батч, фото), ТН из поля, из файла `.txt` или из массива `TAB_NUMS`; скачивание фото через Blob, при блокировке автоскачивания — ссылки «Скачать» на панели. Дополнительно: базовый скрипт с длинным именем (только `TAB_NUMS`) и `_from_file.js` (упрощённая панель).
  - Выгрузка файлов gamification: **`File_DB_Load_GP.js`** — POST `file-download`, панель, группы «Рейтинг» и «Заказы», опционально скользящий старт пакетов; при HTTP 200 и JSON с `success: false` файл не сохраняется.
  - Адресная книга: **`AddressBook_export.js`** — панель в стиле профилей, лог на экране, GET `empInfoFull`, POST `employees/search` (ТН / ФИО), см. [Docs/Скрипт_AddressBook_export.md](Docs/Скрипт_AddressBook_export.md).
  - Турниры: **`Tournament_LeadersForAdmin.js`** — GET `leadersForAdmin`; панель: кнопки «сразу выгрузка» (массив в скрипте, текст в поле, файл `.txt`), два блока CSV (колонки A/B) с настраиваемыми именами полей код/статус, фильтр чекбоксами и **только выбор CSV с диска**; пропуски с 0 лидеров дублируются в консоль. См. [Docs/Скрипт_турниры_leadersForAdmin.md](Docs/Скрипт_турниры_leadersForAdmin.md).
- **Docs/** — подробная документация по отдельным скриптам.
- **log/** — каталог для логов (при наличии логирования в проекте).
- **POST/** — каталог **в `.gitignore`**, в репозиторий **не коммитится**. После клонирования или обновления скриптов выполните `./post_txt_sync.sh` — в `POST/` появятся копии: каждый `Script/*.js` → `POST/<имя>.js.txt`, каждый `Docs/*.md` → `POST/<имя>.md.txt`, `README.md` → `POST/README.md.txt`. Удобно для вставки в системы, где файл должен иметь расширение `.txt`. Команда: `chmod +x post_txt_sync.sh && ./post_txt_sync.sh` из корня репозитория.
- **post_txt_sync.sh** — скрипт пересборки каталога `POST/` (файл в репозитории; сам `POST/` — нет).

## Скрипт выгрузки файлов gamification (`File_DB_Load_GP.js`)

**Задача:** инициировать скачивание выгрузок (CSV и др.) с API gamification в браузере пользователя, используя сессию текущей вкладки (`credentials: "include"`), без вшивания куков и табельных номеров.

**Решение:** фиксированный список задач (основные три запроса + 10 вариантов рейтинга + 10 вариантов заказов); для каждой задачи — `fetch` POST с JSON-телом; ответ сохраняется как файл. Пакетные режимы идут **последовательно** с настраиваемой паузой. Панель: у каждой задачи чекбокс и кнопка; группы «Рейтинг» / «Заказы» в двух колонках с кнопкой «все файлы группы»; «Отметить всё» / «Снять отметки»; «Скачать выделенное» по отмеченным чекбоксам; «Закрыть». Полный список без панели — функция `downloadAllJobs()` из консоли.

**Документация:** полное ТЗ по константам, задачам, функциям, UI и логам консоли — в файле  
[Docs/Скрипт_выгрузка_файлов_gamification_File_DB_Load_GP.md](Docs/Скрипт_выгрузка_файлов_gamification_File_DB_Load_GP.md).

**Краткий перечень сущностей в коде:**

| Имя | Назначение |
|-----|------------|
| `DEFAULT_FILE_DL_STAND`, `FILE_DL_ACTIVE_STAND`, `STAND_ORIGINS` | Стенд по умолчанию, выбор на панели (список), таблица origin |
| `DOWNLOAD_JOBS` | Три основные выгрузки (турниры, награды, статистика) |
| `RATING_GROUP_JOBS` | 10 POST на рейтинг (`businessBlock` + `timePeriod`) |
| `ORDERS_GROUP_JOBS` | 10 POST на заказы (`businessBlock` + `listType`, в т.ч. `NONSEASON`) |
| `getAllDownloadJobs`, `downloadOneJob`, `downloadJobsSequentially`, `downloadJobsStaggered`, `downloadJobsBatch` | Сборка списка, один запрос, пакет последовательно или со скользящим стартом |
| `downloadCheckedPanelJobs`, `downloadRatingGroupOnly`, `downloadOrdersGroupOnly` | Пакеты с панели (режим stagger по чекбоксу) |
| `downloadAllJobs` | Все задачи подряд (вызов из консоли, отдельной кнопки на панели нет) |
| `startDownloadPanel` | Отрисовка панели (автовызов при вставке скрипта) |

**Запуск:** открыть нужный стенд в браузере → DevTools → Console → вставить содержимое `Script/File_DB_Load_GP.js` → Enter.

## Скрипт загрузки профиля героев

**Задача:** пакетно запросить профили по табельным номерам через API геймификации и сохранить ответы в JSON на диск пользователя.

**Решение (основной файл `Profile_GP_LOAD_file.js`):** IIFE, панель без перезагрузки страницы; POST с куками для SIGMA; опционально вырезание base64 фото из JSON (`photoDataInfo`) и сохранение снимков в файлы; поиск контейнера `photoData` / `photoDataKpk` во вложенной структуре `body`; декодирование base64 с padding и URL-safe; ручные ссылки на панели, если браузер блокирует программное скачивание после `fetch`.

**Запуск:** страница стенда → консоль → вставить `Script/Profile_GP_LOAD_file.js`.

Полный перечень констант, функций и сценариев — в [Docs/Скрипт_загрузка_профиля_герои.md](Docs/Скрипт_загрузка_профиля_герои.md). В документации не приводятся полные URL и реальные табельные номера; используются маски и примеры вида `00001234`.
