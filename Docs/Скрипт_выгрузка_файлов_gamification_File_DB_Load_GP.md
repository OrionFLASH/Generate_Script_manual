# Скрипт выгрузки файлов gamification (`File_DB_Load_GP.js`)

## 1. Назначение и задача

**Задача:** в браузере, в контексте уже открытой сессии на стенде, последовательно или по одному запросить у backend выгрузки (CSV и др.) через POST-эндпоинты вида `.../file-download`, получить бинарный ответ и инициировать скачивание файла на диск пользователя.

**Технические ограничения:**

- Скрипт вставляется в **консоль разработчика** (DevTools) на странице нужного стенда.
- Авторизация не вшивается в код: используется `fetch` с `credentials: "include"` — уходят **куки текущей вкладки**.
- Заголовки `Origin` и `Referer` из JS задать нельзя; браузер подставляет их сам. Поэтому для корректного поведения прокси/бэкенда рекомендуется открыть вкладку на разделе, соответствующем типу выгрузки (см. `refererPath` у задач).
- Табельные номера и персональные данные в скрипт **не передаются**.
- Весь исполняемый код скрипта находится в **IIFE** (см. § 2): глобальная область не засоряется `const`/`let`, допускается повторная вставка полного файла без перезагрузки вкладки.

## 2. Расположение и запуск

- Файл: `Script/File_DB_Load_GP.js`.
- **Запуск:** скопировать содержимое файла, вставить в консоль на странице выбранного стенда и выполнить (Enter). Сразу отрисуется плавающая панель с кнопками.
- Весь рабочий код обёрнут в **IIFE** — **повторная вставка полного файла** в ту же консоль **без перезагрузки вкладки** не даёт ошибки повторного объявления `const`/`let` в глобальной области. Перед открытием новой панели старая с тем же `id` удаляется; **«Закрыть»** снимает ссылки на DOM и сбрасывает рабочие переменные (`fileDlDetachPanelAndResetRuntime`).

## 3. Выбор окружения (стенд и контур)

| Константа | Значение | Смысл |
|-----------|----------|--------|
| `DEFAULT_FILE_DL_STAND` | `"PROM"` или `"PSI"` | Стенд по умолчанию до открытия панели. |
| `DEFAULT_FILE_DL_CONTOUR` | `"ALPHA"` или `"SIGMA"` | Контур по умолчанию до открытия панели. |
| `FILE_DL_ACTIVE_STAND` | то же | Текущий стенд: задаётся **выпадающим списком** на панели. |
| `FILE_DL_ACTIVE_CONTOUR` | то же | Текущий контур: задаётся **выпадающим списком** на панели. |

Базовые URL задаются в объекте **`STAND_ORIGINS`** как таблица `STAND -> CONTOUR -> origin` (в коде скрипта). Все пути API ниже **дописываются к выбранному origin** без второго слэша в начале пути.

Перед запуском удобнее выбрать на панели тот же стенд/контур, что и сайт во вкладке (куки и Referer). Запросы идут на хост из `STAND_ORIGINS` по выбранной паре.

## 4. Общая схема работы

1. Пользователь нажимает кнопку на панели: одна задача, вся группа «Рейтинг» / «Заказы», либо отмечает чекбоксы и жмёт **«Скачать выделенное»**.
2. Для каждой задачи формируется URL `origin + job.apiPath`, тело POST — JSON из `job.body`.
3. При **HTTP 200** и заголовке **`Content-Type: application/json`** тело разбирается как JSON. Если пришло **`success: false`** и объект **`error`** (например таймаут `PROCESSING_TIMEOUT`) — **файл не сохраняется**, детали (`code`, `text`, `system`, при наличии `uuid`) пишутся в **«Журнал работы»** на панели.
4. Если ответ не JSON-ошибка, тело читается как `Blob`, имя файла: **`job.fileName`** → **Content-Disposition** → имя по `label`/`id` + `.bin`.
5. Скачивание инициируется через `URL.createObjectURL` и клик по `<a download="...">`.

## 6. Наборы задач

### 6.1. `DOWNLOAD_JOBS` — основные выгрузки (3 кнопки)

| id | Суть | apiPath (логически) | body | refererPath (подсказка) |
|----|------|---------------------|------|-------------------------|
| `tournamentListCsv` | Список турниров | tournaments/file-download | `{}` | `/tournaments/list` |
| `employeeRewardsSummary` | Кнопка **«Награды: (LIST REWARD)»** | employee-rewards/file-download | `{ dateFrom }` подставляется в **`downloadOneJob`** из панели | `/awards/list` |
| `administrationStatisticCsv` | Кнопка **«Посещения»** | administration/statistic/file-download | `{}` | `/admin/statistic` |

У этих задач `fileName` обычно `null` — имя берётся с сервера.

### 6.2. `RATING_GROUP_JOBS` — рейтинг (10 файлов)

- Один endpoint: `ratinglist/file-download`.
- Тело: **`businessBlock`** (`KMKKSB` или `MNS`) + **`timePeriod`** (например `ACTIVESEASON`, `SEASON_2025_2`, `ALLTHETIME` и т.д.).
- У каждой задачи задано уникальное **`fileName`**, иначе браузер перезаписывал бы одинаковые ответы с именем вроде `gamification-ratingList.csv`.
- Рекомендуемая страница вкладки: путь содержит **`/rating`**.

### 6.3. `ORDERS_GROUP_JOBS` — заказы (10 файлов)

- Endpoint: `orders/file-download`.
- Тело: **`businessBlock`** + **`listType`** (не `timePeriod`).
- По смыслу набор периодов **аналогичен** рейтингу, но вместо двух запросов с «активным сезоном» используются два с **`listType: "NONSEASON"`** (для KMKKSB и MNS).
- Уникальные **`fileName`** по той же причине, что и для рейтинга.
- Рекомендуемая страница: **`/admin/orders`**.

### 6.4. Порядок задач на панели и в «Скачать выделенное»

Функция **`getAllDownloadJobs()`** возвращает: **основные** → **рейтинг** → **заказы** (23 задачи при текущей конфигурации). Тот же порядок используется при обходе чекбоксов сверху вниз: сначала основные, затем рейтинг, затем заказы.

Кнопки **«Отметить всё»** / **«Снять отметки»** действуют на все чекбоксы панели.

## 7. Панель управления (UI)

После выполнения скрипта в правом верхнем углу появляется блок (ширина до **`min(960px, 100vw − 16px)`**, высота ограничена **`100vh − 16px`** с вертикальной прокруткой; стиль в одном ключе с **AddressBook_export** / **Tournament_LeadersForAdmin**; кнопки строк задач низкие, с градиентами и `border-radius: 8px`).

1. Заголовок и краткая строка-сводка (число основных, рейтинга, заказов, всего задач с чекбоксом и **динамически** **`(отмечено: X)`** — сколько чекбоксов включено; обновляется при каждом переключении и по кнопкам «Отметить всё» / «Снять отметки»).
2. **Стенд и контур** — два выпадающих списка (`PROM/PSI` и `ALPHA/SIGMA`, хосты из `STAND_ORIGINS`); **справа в той же строке** — **«Отметить всё»** и **«Снять отметки»** (вторичный стиль).
3. **Пакет: паузы** — одна строка внутри карточки (подписи могут переноситься): заголовок **«ПАКЕТ: ПАУЗЫ»**, поля **«Пауза между файлами пакета (после ответа), мс»** и **«Мин. интервал между стартами POST (при перекрытии), мс»** (значения по умолчанию — `DOWNLOAD_ALL_DELAY_MS` и `DOWNLOAD_STAGGER_MS`), чекбокс **«Перекрывать запросы в пакете (скользящий старт)»**; у блока, полей и чекбокса — **`title`** с деталями; **справа** — **`Награды с:`** и **`input type="date"`**. `dateFrom` в POST наград; при пустой дате — **`DEFAULT_EMPLOYEE_REWARDS_DATE_FROM`**. **`syncFileDlDelaysFromPanel()`** / **`syncFileDlRewardsDateFromPanel()`**. Одиночный клик по кнопке задачи паузы **не** использует.
4. **Основные выгрузки** — карточка с голубым градиентом (как визуальный уровень колонок ниже): сетка **из трёх колонок в один ряд**, в каждой ячейке **чекбокс** + кнопка скачивания этой задачи.
5. **Две колонки в один ряд:** слева **Рейтинг**, справа **Заказы**; у каждой задачи чекбокс + кнопка, внизу колонки — кнопка загрузки **всех** файлов группы (без привязки к чекбоксам).
6. **Скачать выделенное (по чекбоксам)** — пакет отмеченных задач. Если ничего не отмечено — предупреждение в **«Журнал работы»** на панели. Режим пакета (последовательно / перекрытие) — чекбоксом **«Перекрывать запросы…»** в блоке пауз.
7. **Журнал работы** — прокрутка внизу (`min-height` / `max-height` чуть больше после переноса кнопок отметок вверх); сообщения через `fileDlPanelEcho`. Начальное состояние: «—».
8. **Закрыть** — **`fileDlDetachPanelAndResetRuntime()`** (обнуление приёмника лога, полей панели, сброс стенда/контура/дат/пауз/флага перекрытия к значениям по умолчанию), затем удаление контейнера с **`id="fileDlGamificationPanelRoot"`** из DOM.

По умолчанию чекбоксы **сняты**. Функции скрипта находятся **внутри IIFE** — из глобальной консоли к **`downloadAllJobs()`** напрямую обратиться нельзя; для повторного открытия панели снова вставьте **весь** файл или закройте панель и вставьте скрипт ещё раз.

Стили заданы инлайн в `startDownloadPanel()`; при нехватке высоты экрана у контейнера включена прокрутка (`max-height`, `overflow-y`).

## 8. Журнал работы и консоль

Сообщения формируются через **`fileDlPanelEcho(level, msg)`**: уровни **`log`** и **`warn`** пишут **только** в блок **«Журнал работы»** на панели (через **`fileDlPanelLogAppend`**). Уровень **`error`** — в журнал и в **`console.error`**. В **консоли** дополнительно: строка при открытии панели; при **одиночной** загрузке — краткий итог (OK / ошибка); при **пакете** — строка в начале и сводка в конце (число задач, успешных, ошибок). Формат текста в журнале **без рамок и без дублирования** полей между стартом и финишем одного запроса.

Вспомогательная функция **`fileDlConsoleSingleJobSummary`** выводит краткий итог одиночного запроса, если в контексте нет порядкового номера в пакете (`index` / `total`).

**Одиночный запрос:**

- В начале: `СТАРТ загрузки` — группа, пакет/режим, при пакете — «Файл в пакете: N из M», `Задача id`.
- При успехе: `ЗАВЕРШЕНО: файл скачан` — **стенд/контур**, **id задачи**, **POST body** (JSON), путь API, имя сохранённого файла, размер в байтах (чтобы при скользящем старте различать параллельные завершения).
- Ошибка сети: `ОШИБКА (сеть / исключение)` + текст исключения.
- Ошибка HTTP: `ОШИБКА HTTP` + статус (без повторного id задачи).
- Ошибка API в JSON при HTTP 200: `ОШИБКА API (HTTP 200, JSON)` и поля `error` (без строки «ЗАВЕРШЕНО: файл скачан»).

**Пакет (`downloadJobsSequentially` / `downloadJobsStaggered`):**

- Старт пакета: название, число файлов, пауза.
- Между файлами: строка о паузе и номере следующего файла.
- Финиш: счётчики успешных и неуспешных.

## 9. Функции и переменные (справочник)

### Константы и массивы

- **`DEFAULT_FILE_DL_STAND`**, **`DEFAULT_FILE_DL_CONTOUR`**, **`FILE_DL_ACTIVE_STAND`**, **`FILE_DL_ACTIVE_CONTOUR`**, **`STAND_ORIGINS`** — пара окружения по умолчанию, текущий выбор с панели и таблица базовых URL.

## 10. Настройка под себя

- Сменить окружение по умолчанию: **`DEFAULT_FILE_DL_STAND`** и **`DEFAULT_FILE_DL_CONTOUR`**; во время работы — списки на панели (`FILE_DL_ACTIVE_STAND`, `FILE_DL_ACTIVE_CONTOUR`).
- Дата наград: на панели поле **«Награды с»**; значение по умолчанию в коде — **`DEFAULT_EMPLOYEE_REWARDS_DATE_FROM`**.
- Пауза между файлами и интервал стартов по умолчанию: **`DOWNLOAD_ALL_DELAY_MS`**, **`DOWNLOAD_STAGGER_MS`** (на панели — длинные подписи к полям).
- Добавить сезон или блок: скопировать элемент в **`RATING_GROUP_JOBS`** или **`ORDERS_GROUP_JOBS`**, поправить `body` и уникальный **`fileName`**.

## 11. Каталог POST/ и имена с суффиксом .txt

В корне репозитория скрипт **`post_txt_sync.sh`** создаёт (или перезаписывает) каталог **`POST/`**, куда копируются:

- каждый файл `Script/*.js` → `POST/<имя>.js.txt`;
- каждый файл `Docs/*.md` → `POST/<имя>.md.txt`;
- `README.md` → `POST/README.md.txt`;
- из корня репозитория: **`.gitignore`** → `POST/.gitignore.txt`, **`post_txt_sync.sh`** → `POST/post_txt_sync.sh.txt`.

Содержимое **`POST/`** в **git не попадает** (запись в `.gitignore`), чтобы не дублировать историю и не плодить расхождения с оригиналами. После правок исходников снова запустите `./post_txt_sync.sh`. Для вставки кода в консоль браузера открывайте исходный `Script/File_DB_Load_GP.js` или его копию в `POST/` — текст одинаковый.
