# Скрипт выгрузки файлов gamification (`File_DB_Load_GP.js`)

## 1. Назначение и задача

**Задача:** в браузере, в контексте уже открытой сессии на стенде, последовательно или по одному запросить у backend выгрузки (CSV и др.) через POST-эндпоинты вида `.../file-download`, получить бинарный ответ и инициировать скачивание файла на диск пользователя.

**Технические ограничения:**

- Скрипт вставляется в **консоль разработчика** (DevTools) на странице нужного стенда.
- Авторизация не вшивается в код: используется `fetch` с `credentials: "include"` — уходят **куки текущей вкладки**.
- Заголовки `Origin` и `Referer` из JS задать нельзя; браузер подставляет их сам. Поэтому для корректного поведения прокси/бэкенда рекомендуется открыть вкладку на разделе, соответствующем типу выгрузки (см. `refererPath` у задач).
- Табельные номера и персональные данные в скрипт **не передаются**.

## 2. Расположение и запуск

- Файл: `Script/File_DB_Load_GP.js`.
- **Запуск:** скопировать содержимое файла, вставить в консоль на странице выбранного стенда и выполнить (Enter). Сразу отрисуется плавающая панель с кнопками.
- При повторной вставке того же кода в одной сессии возможна ошибка повторного объявления `const` — обновите страницу или вставляйте скрипт один раз.

## 3. Выбор стенда

| Константа | Значение | Смысл |
|-----------|----------|--------|
| `DEFAULT_FILE_DL_STAND` | `"SIGMA"` или `"ALPHA"` | Значение по умолчанию до открытия панели. |
| `FILE_DL_ACTIVE_STAND` | то же | Текущий стенд: задаётся **выпадающим списком** на панели; от него берётся origin для POST. |

Базовые URL задаются в объекте **`STAND_ORIGINS`** (в коде скрипта). Все пути API ниже **дописываются к выбранному origin** без второго слэша в начале пути.

Перед запуском удобнее выбрать на панели тот же стенд, что и сайт во вкладке (куки и Referer). Запросы идут на хост из `STAND_ORIGINS` по выбранному значению.

## 4. Общая схема работы

1. Пользователь нажимает кнопку на панели: одна задача, вся группа «Рейтинг» / «Заказы», либо отмечает чекбоксы и жмёт **«Скачать выделенное»**.
2. Для каждой задачи формируется URL `origin + job.apiPath`, тело POST — JSON из `job.body`.
3. При **HTTP 200** и заголовке **`Content-Type: application/json`** тело разбирается как JSON. Если пришло **`success: false`** и объект **`error`** (например таймаут `PROCESSING_TIMEOUT`) — **файл не сохраняется**, в консоль выводится `code`, `text`, `system`, при наличии `uuid`.
4. Если ответ не JSON-ошибка, тело читается как `Blob`, имя файла: **`job.fileName`** → **Content-Disposition** → имя по `label`/`id` + `.bin`.
5. Скачивание инициируется через `URL.createObjectURL` и клик по `<a download="...">`.

## 5. Константы конфигурации

| Имя | Назначение |
|-----|------------|
| `DEFAULT_FILE_DOWNLOAD_PATH` | Путь по умолчанию для задач без своего `apiPath` (турниры). |
| `EMPLOYEE_REWARDS_DATE_FROM` | Поле `dateFrom` в теле запроса сводки наград. |
| `DOWNLOAD_ALL_DELAY_MS` | Пауза в миллисекундах **между** запросами при последовательной пакетной загрузке; также минимальная пауза после **успеха** предыдущего запроса в режиме скользящего старта. |
| `DOWNLOAD_STAGGER_MS` | В режиме скользящего старта: минимальный интервал между **запусками** POST (следующий старт не раньше, чем через это время, если предыдущий ещё не завершился успешно с короткой паузой). |
| `FILE_DL_USE_STAGGER` | Флаг с панели (чекбокс): включён ли скользящий старт для пакетов с панели. |
| `RATINGLIST_FILE_DOWNLOAD_PATH` | Общий путь для всех задач группы «Рейтинг». |
| `ORDERS_FILE_DOWNLOAD_PATH` | Общий путь для всех задач группы «Заказы». |

## 6. Наборы задач

### 6.1. `DOWNLOAD_JOBS` — основные выгрузки (3 кнопки)

| id | Суть | apiPath (логически) | body | refererPath (подсказка) |
|----|------|---------------------|------|-------------------------|
| `tournamentListCsv` | Список турниров | tournaments/file-download | `{}` | `/tournaments/list` |
| `employeeRewardsSummary` | Сводка наград | employee-rewards/file-download | `{ dateFrom }` | `/awards/list` |
| `administrationStatisticCsv` | Статистика админки | administration/statistic/file-download | `{}` | `/admin/statistic` |

У этих задач `fileName` обычно `null` — имя берётся с сервера.

### 6.2. `RATING_GROUP_JOBS` — рейтинг (10 файлов)

- Один endpoint: `ratinglist/file-download`.
- Тело: **`businessBlock`** (`KMKKSB` или `MNS`) + **`timePeriod`** (например `ACTIVESEASON`, `SEASON_2025_2`, `ALLTHETIME` и т.д.).
- У каждой задачи задано уникальное **`fileName`**, иначе браузер перезаписывал бы одинаковые ответы с именем вроде `gamification-ratingList.csv`.
- Рекомендуемая страница вкладки: путь содержит **`/rating`**.

### 6.3. `ORDERS_GROUP_JOBS` — заказы (10 файлов)

- Endpoint: `orders/file-download`.
- Тело: **`businessBlock`** + **`listType`** (не `timePeriod`).
- По смыслу набор периодов **аналогичен** рейтингу, но вместо двух запросов с «активным сезоном» используются два с **`listType: "NONSEASON"`** (для KMKKSB и MNS).
- Уникальные **`fileName`** по той же причине, что и для рейтинга.
- Рекомендуемая страница: **`/admin/orders`**.

### 6.4. Порядок задач на панели и в «Скачать выделенное»

Функция **`getAllDownloadJobs()`** возвращает: **основные** → **рейтинг** → **заказы** (23 задачи при текущей конфигурации). Тот же порядок используется при обходе чекбоксов сверху вниз: сначала основные, затем рейтинг, затем заказы.

Кнопки **«Отметить всё»** / **«Снять отметки»** действуют на все чекбоксы панели.

## 7. Панель управления (UI)

После выполнения скрипта в правом верхнем углу появляется блок:

1. Заголовок и краткая строка-сводка (число основных, рейтинга, заказов, всего задач с чекбоксом, значение `dateFrom` для наград).
2. **Стенд** — выпадающий список ALPHA / SIGMA (базовый URL из `STAND_ORIGINS`).
3. Чекбокс **«Пакеты: скользящий старт»** — для кнопок «все файлы группы» и «Скачать выделенное» задаёт режим `downloadJobsStaggered` вместо строго последовательного.
4. **Основные выгрузки** — у каждой задачи строка: **чекбокс** + кнопка скачивания этой задачи.
5. **Две колонки в один ряд:** слева **Рейтинг**, справа **Заказы**; у каждой задачи чекбокс + кнопка, внизу колонки — кнопка загрузки **всех** файлов группы (без привязки к чекбоксам).
6. **Отметить всё** / **Снять отметки** — мелкие кнопки, управляют всеми чекбоксами панели.
7. **Скачать выделенное (по чекбоксам)** — пакет отмеченных задач (последовательно или скользящий старт по чекбоксу выше). Если ничего не отмечено — предупреждение в консоль.
8. **Закрыть** — удаляет панель со страницы.

По умолчанию чекбоксы **сняты**; функция **`downloadAllJobs()`** в коде сохранена для вызова из консоли вручную при необходимости.

Стили заданы инлайн в `startDownloadPanel()`; при нехватке высоты экрана у контейнера включена прокрутка (`max-height`, `overflow-y`).

## 8. Логирование в консоль

Формат **без рамок и без дублирования** полей между стартом и финишем одного запроса.

**Одиночный запрос:**

- В начале: `СТАРТ загрузки` — группа, пакет/режим, при пакете — «Файл в пакете: N из M», `Задача id`.
- При успехе: `ЗАВЕРШЕНО: файл скачан` — **стенд**, **id задачи**, **POST body** (JSON), путь API, имя сохранённого файла, размер в байтах (чтобы при скользящем старте различать параллельные завершения).
- Ошибка сети: `ОШИБКА (сеть / исключение)` + текст исключения.
- Ошибка HTTP: `ОШИБКА HTTP` + статус (без повторного id задачи).
- Ошибка API в JSON при HTTP 200: `ОШИБКА API (HTTP 200, JSON)` и поля `error` (без строки «ЗАВЕРШЕНО: файл скачан»).

**Пакет (`downloadJobsSequentially` / `downloadJobsStaggered`):**

- Старт пакета: название, число файлов, пауза.
- Между файлами: строка о паузе и номере следующего файла.
- Финиш: счётчики успешных и неуспешных.

## 9. Функции и переменные (справочник)

### Константы и массивы

- **`DEFAULT_FILE_DL_STAND`**, **`FILE_DL_ACTIVE_STAND`**, **`STAND_ORIGINS`** — стенд по умолчанию, текущий выбор с панели и таблица базовых URL.
- **`DOWNLOAD_JOBS`**, **`RATING_GROUP_JOBS`**, **`ORDERS_GROUP_JOBS`** — описания задач (см. выше).

### Функции

| Функция | Назначение |
|---------|------------|
| `getAllDownloadJobs()` | Объединяет все задачи (основные + рейтинг + заказы); используется в коде и удобна для ручного вызова из консоли. |
| `getGroupNameForJob(job)` | Имя группы для логов по `apiPath` (Рейтинг, Заказы, Турниры и т.д.). |
| `delay(ms)` | Промис с таймаутом для паузы между запросами. |
| `parseFilenameFromContentDisposition(header)` | Разбор имени файла из `Content-Disposition` (в т.ч. UTF-8 `filename*`). |
| `downloadOneJob(job, ctx?)` | Один POST + скачивание; учёт JSON-ошибки при 200. Возвращает `{ ok, fileName?, status?, error?, apiError? }`. |
| `formatPostBodyForLog(bodyObj)` | Строка JSON тела POST для логов. |
| `isJsonContentType(ct)` | Проверка заголовка Content-Type на `application/json`. |
| `downloadJobsSequentially(jobs, logLabel)` | Строго по одному запросу, пауза `DOWNLOAD_ALL_DELAY_MS`. |
| `downloadJobsStaggered(jobs, logLabel)` | Перекрывающиеся запросы: `Promise.race` между `DOWNLOAD_STAGGER_MS` и успехом предыдущего + `DOWNLOAD_ALL_DELAY_MS`. |
| `downloadJobsBatch(jobs, logLabel, useStagger)` | Обёртка: последовательно или скользящий старт. |
| `downloadAllJobs()` | Все задачи из `getAllDownloadJobs()` (всегда последовательно; вызов из консоли). |
| `downloadCheckedPanelJobs(entries)` | Отмеченные чекбоксами; режим пакета по `FILE_DL_USE_STAGGER`. |
| `downloadRatingGroupOnly()` | Вся группа рейтинга; режим по `FILE_DL_USE_STAGGER`. |
| `downloadOrdersGroupOnly()` | Вся группа заказов; режим по `FILE_DL_USE_STAGGER`. |
| `startDownloadPanel()` | Создаёт DOM-панель и вешает обработчики. |
| *(автовызов)* `startDownloadPanel()` в конце файла | Панель показывается сразу после вставки скрипта. |

**Пример сценария без панели (теоретически):** после загрузки скрипта можно было бы вызвать в консоли `downloadAllJobs()` — если убрать автозапуск панели, но в текущей версии панель открывается автоматически; повторные вызовы функций доступны из консоли, пока страница не перезагружена.

## 10. Настройка под себя

- Сменить стенд по умолчанию: **`DEFAULT_FILE_DL_STAND`**; во время работы — список на панели (`FILE_DL_ACTIVE_STAND`).
- Изменить дату наград: **`EMPLOYEE_REWARDS_DATE_FROM`**.
- Изменить паузу пакета: **`DOWNLOAD_ALL_DELAY_MS`**; интервал скользящего старта: **`DOWNLOAD_STAGGER_MS`**.
- Добавить сезон или блок: скопировать элемент в **`RATING_GROUP_JOBS`** или **`ORDERS_GROUP_JOBS`**, поправить `body` и уникальный **`fileName`**.

## 11. Каталог POST/ и имена с суффиксом .txt

В корне репозитория скрипт **`post_txt_sync.sh`** создаёт (или перезаписывает) каталог **`POST/`**, куда копируются:

- каждый файл `Script/*.js` → `POST/<имя>.js.txt`;
- каждый файл `Docs/*.md` → `POST/<имя>.md.txt`;
- `README.md` → `POST/README.md.txt`.

Содержимое **`POST/`** в **git не попадает** (запись в `.gitignore`), чтобы не дублировать историю и не плодить расхождения с оригиналами. После правок исходников снова запустите `./post_txt_sync.sh`. Для вставки кода в консоль браузера открывайте исходный `Script/File_DB_Load_GP.js` или его копию в `POST/` — текст одинаковый.
