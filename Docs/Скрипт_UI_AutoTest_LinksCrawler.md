# Скрипт многоэтапного обхода ссылок (`UI_AutoTest_LinksCrawler.js`)

## 1. Назначение

Скрипт запускается в консоли браузера на текущей странице и выполняет поэтапный обход внутренних ссылок сайта.

Ключевые задачи:

- собрать ссылки этапа 1 со стартовой страницы;
- дать вручную выбрать, какие ссылки запускать на каждом этапе;
- переходить по ссылкам только кликом по `<a>`, а не через прямую подмену `location`;
- строить следующий этап из дочерних ссылок, найденных на успешных переходах;
- сохранять статусы переходов (`OK`, `FAIL`, `SKIPPED`) и выводить дерево результатов;
- поддерживать остановку и диагностику падений.

---

## 2. Запуск

1. Откройте нужную страницу приложения.
2. Вставьте полный файл `Script/UI_AutoTest_LinksCrawler.js` в DevTools Console.
3. Нажмите Enter.

Скрипт обёрнут в IIFE и удаляет предыдущую панель перед повторным запуском.

---

## 3. Общий сценарий работы

1. **Этап 1:**
   - кнопка `Скан этапа` собирает ссылки текущей страницы;
   - пользователь отмечает ссылки;
   - кнопка `Запуск этапа` запускает переходы по выбранным ссылкам.
2. **Этапы 2..N:**
   - ссылки формируются автоматически из дочерних ссылок успешных узлов предыдущего этапа;
   - пользователь отмечает нужные ссылки и запускает этап.
3. После выполнения этапа:
   - заполняются статусы;
   - формируется набор для следующего этапа;
   - в консоль выводится дерево состояния.

Во время выполнения панель сворачивается в компактный режим с кнопкой остановки.

---

## 4. Логика переходов и контекст

- Переход выполняется через поиск DOM-ссылки по абсолютному URL и вызов `a.click()`.
- Ожидание после перехода контролируется `waitPageReadyWithMinDelay(...)` и паузой этапа.
- Для этапов с вложенностью учитывается путь контекста (`contextPath`).
- На этапе 1 (пустой контекст) восстановление через `history.back()` не выполняется.
- На подэтапах ссылки группируются по родителю:
  - внутри группы возврат идёт только на страницу родителя `-1`;
  - переход к контексту выше (`-2` и далее) выполняется только при смене родителя.

---

## 5. Остановка и сброс

### 5.1 Остановка во время выполнения этапа

- прерывает текущий цикл;
- выводит в консоль дерево **только проблемных** ссылок (ошибки/предупреждения) по пройденным этапам;
- оставляет текущий UI-срез, чтобы можно было проанализировать результаты.

### 5.2 Остановка между этапами

- сразу выводит дерево проблемных ссылок;
- затем выполняет полный сброс состояния к старту:
  - этап = 1;
  - очищаются узлы, связи, найденные дочерние ссылки и выбранные элементы.

---

## 6. Логирование и диагностика

### 6.1 Основной лог

- рабочие события пишутся в консоль с префиксом `[UI_LinksCrawler]`.

### 6.2 Выбор транспорта логирования (авто по браузеру)

- чекбокс `Лог в файл (онлайн)` включает дополнительный транспорт;
- если браузер поддерживает File System Access API и выбран файл: режим `file`;
- если API недоступен (например Safari) или файл не выбран: режим `sessionStorage`.

При активации выводится служебная строка:

- `[UI_LinksCrawler][LOG_MODE] file`
- или `[UI_LinksCrawler][LOG_MODE] sessionStorage`

### 6.3 Аварийный лог

- последние строки также буферизуются в `sessionStorage`;
- при повторном запуске выводятся в консоль блоком `[RESTORE]`;
- это помогает диагностировать место падения, если страница перезагрузилась или панель исчезла.

---

## 7. Ограничения на объём данных

Для защиты от перегрузки на глубоких обходах добавлены настраиваемые лимиты:

- `Лимит ссылок/этап`
- `Лимит детей/родитель`
- `Рендер этап 1`
- `Рендер группа`

Также есть кнопка `↺ Сброс лимитов` для возврата к дефолтным значениям.

Если лимит достигнут, это фиксируется в логе, а UI показывает срез `Показано X из Y`.

---

## 8. Основные структуры и статусы

### 8.1 LinkNode

Каждая ссылка хранится как узел с полями:

- `key`, `href`, `depth`, `parentKey`;
- `contextPath`, `selected`;
- `status`, `reason`;
- `fromUrl`, `toUrl`, `elapsedMs`.

### 8.2 Статусы

- `OK` — переход выполнен успешно;
- `FAIL` — ошибка перехода/контекста/загрузки;
- `SKIPPED` — ссылка не была отмечена к запуску.

---

## 9. Ключевые функции

| Функция | Назначение |
|---|---|
| `stageScanCurrentPage` | Скан ссылок этапа 1 |
| `runStage` | Основной цикл выполнения этапа |
| `openContextPath` | Восстановление контекста перед переходом |
| `ensureBackToUrl` | Возврат назад до целевого URL |
| `collectLinksOnCurrentPage` | Сбор уникальных ссылок текущей страницы |
| `renderStageLinks` | Отрисовка ссылок этапа и групп по родителю |
| `appendTreeSummaryToLog` | Вывод дерева полного/проблемного результата |
| `resetCrawlerStateToStart` | Полный сброс состояния к этапу 1 |
| `rebuildLimitsFromInput` | Применение лимитов из UI |
| `pickLogFile` | Выбор файла для режима `file` |

---

## 10. Ограничения и особенности

- Скрипт работает в контексте текущей вкладки и зависит от реального DOM.
- Для режима онлайн-записи в файл требуется поддержка File System Access API.
- В Safari рекомендуется использовать режим `sessionStorage` + восстановление `[RESTORE]`.
- Переходы выполняются только по найденным элементам `<a>` на странице.
