# Скрипт выгрузки `leadersForAdmin` по турнирам (`Tournament_LeadersForAdmin.js`)

## 1. Назначение

На странице стенда gamification (ALPHA / SIGMA) из консоли выполняются GET-запросы вида:

`{TOURNAMENT_BASE}/{tournamentId}/leadersForAdmin?pageNum=1`

Ответы JSON собираются в **один** файл: объект, ключи — коды турниров, значения — массив из одного элемента (тело ответа API), чтобы сохранить исходную структуру. Турниры с **нулём лидеров** в ответе пропускаются (не попадают в файл); по каждому такому коду в **консоль** пишется строка `[пропуск: 0 лидеров в ответе]`, в конце — сводный `console.log` со списком кодов. Опционально из дерева удаляется **`photoData`**.

## 2. Запуск

- Файл: `Script/Tournament_LeadersForAdmin.js`.
- Вкладка на нужном стенде → Console → вставить скрипт → Enter.
- Скрипт обёрнут в **IIFE**: повторная вставка в ту же вкладку не вызывает `SyntaxError` на повторном `const`. Старая панель с `id="tournamentLeadersForAdminRoot"` перед показом новой удаляется.
- Стенд выбирается **выпадающим списком** на панели (`TOURNAMENT_UI_STAND`); базовый URL берётся из **`TOURNAMENT_BASE`**. Удобнее совпадение с вкладкой (куки).

## 3. Константы

| Имя | Назначение |
|-----|------------|
| `DEFAULT_TOURNAMENT_STAND`, `TOURNAMENT_UI_STAND` | Значение по умолчанию и текущий выбор стенда на панели |
| `TOURNAMENT_BASE` | Полные префиксы URL API турниров для ALPHA и SIGMA |
| `TOURNAMENT_IDS_IN_SCRIPT` | Коды по умолчанию (режим «массив в скрипте») |
| `LEADERS_SERVICE` | Сегмент пути после id (по умолчанию `leadersForAdmin`) |
| `REQUEST_GAP_MS` | Пауза между запросами в цикле |
| `STRIP_PHOTO_DATA_DEFAULT` | Начальное состояние чекбокса удаления `photoData` |
| `CSV1_STATUS_LABELS` | Подписи статусов для CSV варианта 1 |
| `CSV2_STATUS_LABELS` | Подписи для CSV варианта 2 |

## 4. Интерфейс панели

- Панель шириной до **920px** (на узком экране колонки CSV складываются в одну под другой).
- **Сверху:** стенд, чекбокс удаления `photoData`, **общее многострочное поле** для простого списка кодов.
- **Кнопки запуска (без радиокнопок):** одно нажатие **сразу** начинает выгрузку. Отдельной кнопки «Запустить» нет.
  - **По массиву в скрипте** — коды из `TOURNAMENT_IDS_IN_SCRIPT`.
  - **По тексту в поле выше** — разбор токенов из общего поля.
  - **Файл .txt — выбрать и сразу выгрузить** — диалог выбора файла → чтение UTF-8 → разбор кодов → выгрузка.
- **Две колонки CSV (A слева, B справа):** в каждой — два **поля ввода** имён колонок (**код** и **статус**, как в 1-й строке CSV), блок **чекбоксов фильтра** по статусу и одна кнопка **«CSV: выбрать файл и сразу выгрузить»** (вставка CSV только через файл, без отдельного текстового поля в колонке). Списки подписей статусов — `CSV1_STATUS_LABELS` / `CSV2_STATUS_LABELS` в коде.
- Внизу — **лог** и **«Закрыть панель»**.
- Пока идёт выгрузка, повторные нажатия блокируются (`exportBusy`).

## 5. Источники кодов (по кнопкам)

1. **Массив в скрипте** — `TOURNAMENT_IDS_IN_SCRIPT`.
2. **Текст в общем поле** — токены `[A-Za-z0-9_-]+`, порядок сохраняется, дубликаты убираются.
3. **Файл .txt** — содержимое выбранного файла, тот же разбор, что и для текста.
4. **CSV колонка A** — левая панель; выбор `.csv`/`.txt` с диска; по умолчанию заголовки **`TOURNAMENT_CODE`** и **`TOURNAMENT_STATUS`** (задаются в полях на панели).
5. **CSV колонка B** — правая панель; то же; по умолчанию **«Код турнира»** и **«Бизнес-статус турнира»**.

Разделитель строк CSV определяется по первой строке: если точек с запятой больше, чем запятых, используется `;`, иначе `,`. Поддерживается простое экранирование полей в кавычках.

## 6. Извлечение лидеров

Число лидеров для решения «пропускать или нет» берётся из:

- `data.body.tournament.leaders`, иначе
- `data.body.badge.leaders`.

Если массива нет или длина 0 — турнир не записывается в итоговый JSON (см. логи пропуска в консоли).

## 7. Функции (кратко)

| Функция | Назначение |
|---------|------------|
| `parseTournamentCodesFromText` | Коды из произвольного текста |
| `parseCsv` | Разбор CSV в заголовки и строки |
| `codesFromCsvByColumns` | Отбор кодов по задаваемым именам колонок (код + статус) и фильтру чекбоксов |
| `fetchLeadersForAdmin` | Один GET |
| `removePhotoData` | Рекурсивное удаление ключей `photoData` |
| `runExport` (внутри панели) | Цикл по кодам, сбор JSON, скачивание файла |
| `createCsvColumn` (внутри панели) | Колонка CSV: поля имён колонок, чекбоксы, кнопка выбора файла и вызов `runExport` |
| `startTournamentPanel` | Сборка UI и обработчики кнопок |

Имя файла: `leadersForAdmin_{STAND}_{YYYYMMDD-HHMMSS}.json`.
