# Скрипт выгрузки `leadersForAdmin` по турнирам (`Tournament_LeadersForAdmin.js`)

## 1. Назначение

На странице стенда gamification (ALPHA / SIGMA) из консоли выполняются GET-запросы вида:

`{TOURNAMENT_BASE}/{tournamentId}/leadersForAdmin?pageNum=1`

Ответы JSON собираются в **один** файл: объект, ключи — коды турниров, значения — массив из одного элемента (тело ответа API), чтобы сохранить исходную структуру. Турниры с **нулём лидеров** в ответе пропускаются (не попадают в файл). Опционально из дерева удаляется **`photoData`**.

## 2. Запуск

- Файл: `Script/Tournament_LeadersForAdmin.js`.
- Вкладка на нужном стенде → Console → вставить скрипт → Enter.
- Скрипт обёрнут в **IIFE**: повторная вставка в ту же вкладку не вызывает `SyntaxError` на повторном `const`. Старая панель с `id="tournamentLeadersForAdminRoot"` перед показом новой удаляется.
- Стенд выбирается **выпадающим списком** на панели (`TOURNAMENT_UI_STAND`); базовый URL берётся из **`TOURNAMENT_BASE`**. Удобнее совпадение с вкладкой (куки).

## 3. Константы

| Имя | Назначение |
|-----|------------|
| `DEFAULT_TOURNAMENT_STAND`, `TOURNAMENT_UI_STAND` | Значение по умолчанию и текущий выбор стенда на панели |
| `TOURNAMENT_BASE` | Полные префиксы URL API турниров для ALPHA и SIGMA |
| `TOURNAMENT_IDS_IN_SCRIPT` | Коды по умолчанию (режим «массив в скрипте») |
| `LEADERS_SERVICE` | Сегмент пути после id (по умолчанию `leadersForAdmin`) |
| `REQUEST_GAP_MS` | Пауза между запросами в цикле |
| `STRIP_PHOTO_DATA_DEFAULT` | Начальное состояние чекбокса удаления `photoData` |
| `CSV1_STATUS_LABELS` | Подписи статусов для CSV варианта 1 |
| `CSV2_STATUS_LABELS` | Подписи для CSV варианта 2 |

## 4. Источники кодов турнира (радиокнопки)

1. **Массив в скрипте** — `TOURNAMENT_IDS_IN_SCRIPT`.
2. **Текст в textarea** — токены `[A-Za-z0-9_-]+`, порядок сохраняется, дубликаты убираются.
3. **Файл .txt** — то же разбор текста после выбора файла (UTF-8).
4. **CSV вариант 1** — колонки **`TOURNAMENT_CODE`** и **`TOURNAMENT_STATUS`**; в выборку попадают строки, чей статус отмечен чекбоксами блока «Фильтр статусов (CSV1)».
5. **CSV вариант 2** — колонки **«Код турнира»** и **«Бизнес-статус турнира»**; фильтр — чекбоксы «CSV2».

Разделитель строк CSV определяется по первой строке: если точек с запятой больше, чем запятых, используется `;`, иначе `,`. Поддерживается простое экранирование полей в кавычках.

## 5. Извлечение лидеров

Число лидеров для решения «пропускать или нет» берётся из:

- `data.body.tournament.leaders`, иначе
- `data.body.badge.leaders`.

Если массива нет или длина 0 — турнир не записывается в итоговый JSON.

## 6. Функции (кратко)

| Функция | Назначение |
|---------|------------|
| `parseTournamentCodesFromText` | Коды из произвольного текста |
| `parseCsv` | Разбор CSV в заголовки и строки |
| `codesFromCsvVariant1`, `codesFromCsvVariant2` | Отбор кодов по колонкам и фильтру статусов |
| `fetchLeadersForAdmin` | Один GET |
| `removePhotoData` | Рекурсивное удаление ключей `photoData` |
| `startTournamentPanel` | UI и запуск выгрузки |

Имя файла: `leadersForAdmin_{STAND}_{YYYYMMDD-HHMMSS}.json`.
