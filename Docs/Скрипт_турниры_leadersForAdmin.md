# Скрипт выгрузки `leadersForAdmin` по турнирам (`Tournament_LeadersForAdmin.js`)

## 1. Назначение

На странице стенда gamification (ALPHA / SIGMA) из консоли выполняются GET-запросы вида:

`{TOURNAMENT_BASE}/{tournamentId}/leadersForAdmin?pageNum=1`

Ответы JSON собираются в **один** файл: ключи — коды турниров, значения — массив из **одного** объекта. Успешный ответ с лидерами пишется **как вернул API**. **0 лидеров** — нормализованная запись `success: false` с `body.tournament` и `contestants: «0 участников»`. **Ошибка** (`success: false` и блок `error`, или HTTP без ожидаемого JSON) — в файле сохраняется объект с **`error`**; в сводке лога помечается как **ERROR**. При HTTP OK и **полностью пустом теле** (`null`) строка в JSON **не** создаётся. Файл **не** скачивается, если нечего записать или итог — только `{}`.

## 2. Запуск

- Файл: `Script/Tournament_LeadersForAdmin.js`.
- Вкладка на нужном стенде → Console → вставить скрипт → Enter.
- Скрипт обёрнут в **IIFE**: повторная вставка в ту же вкладку не вызывает `SyntaxError` на повторном `const`. Старая панель с `id="tournamentLeadersForAdminRoot"` перед показом новой удаляется.
- Стенд выбирается **выпадающим списком** на панели (`TOURNAMENT_UI_STAND`); базовый URL берётся из **`TOURNAMENT_BASE`**. Удобнее совпадение с вкладкой (куки).

## 3. Константы

| Имя | Назначение |
|-----|------------|
| `DEFAULT_TOURNAMENT_STAND`, `TOURNAMENT_UI_STAND` | Значение по умолчанию и текущий выбор стенда на панели |
| `TOURNAMENT_BASE` | Полные префиксы URL API турниров для ALPHA и SIGMA |
| `LEADERS_SERVICE` | Сегмент пути после id (по умолчанию `leadersForAdmin`) |
| `DEFAULT_REQUEST_GAP_MS` | Значение по умолчанию в поле «Пауза, мс» на панели (диапазон 0–60000) |
| `DEFAULT_EXPORT_FILENAME_PREFIX_PLACEHOLDER` | Подсказка в поле префикса имени файла (пустое значение = авто `leadersForAdmin_{стенд}_`) |
| `CSV1_STATUS_LABELS` | Подписи статусов для CSV варианта 1 |
| `CSV2_STATUS_LABELS` | Подписи для CSV варианта 2 |
| `CSV1_DEFAULT_CHECKED_STATUSES`, `CSV2_DEFAULT_CHECKED_STATUSES` | Какие статусы **включены в фильтре по умолчанию** при открытии панели (остальные чекбоксы сняты); сейчас только активный: `АКТИВНЫЙ` / `Активный` |
| `CSV_CODE_COLUMN_PRESETS`, `CSV_STATUS_COLUMN_PRESETS` | Варианты в `<select>` для имён колонок кода и статуса в общем блоке CSV |
| `DEFAULT_TOURNAMENT_CODES_TEXTAREA` | Коды по умолчанию в общем многострочном поле при открытии панели |

## 4. Интерфейс панели

- Панель шириной до **920px**, высота до **92vh**; корень — колонка **flex** с `overflow: hidden`, средняя часть (поля и CSV) в **своей** прокрутке, блок лога — **фиксированной высоты** с прокруткой **только внутри лога** (страница под панелью не «уезжает» из‑за ленты).
- **Сверху:** строка на всю ширину — **стенд** слева; **справа** — поле **«Префикс имени файла (до даты)»** (часть имени до таймштампа; пусто = `leadersForAdmin_{ALPHA|SIGMA}_`) и **«Пауза, мс»** между GET. Ниже — **общее многострочное поле** кодов (по умолчанию предзаполнено списком `DEFAULT_TOURNAMENT_CODES_TEXTAREA`, можно заменить).
- **Кнопки запуска (без радиокнопок):** одно нажатие **сразу** начинает выгрузку. Отдельной кнопки «Запустить» нет. Две кнопки **в один ряд**:
  - **По тексту в поле выше** — разбор токенов из общего поля.
  - **Файл .txt — выбрать и сразу выгрузить** — диалог выбора файла → чтение UTF-8 → разбор кодов → выгрузка.
- **Блок CSV «Загрузка из TOURNAMENT_SHEDULE / LIST»:** сверху один раз — два **`<select>`** с пресетами имён колонок + «Другой заголовок…»; эти имена используют **обе** кнопки выгрузки. Ниже — **два отдельных** набора чекбоксов статусов: у кнопки SHEDULE учитывается только левый фильтр, у кнопки LIST — только правый (`CSV1_STATUS_LABELS` / `CSV2_STATUS_LABELS`).
- Внизу — **лог-лента** (строки с меткой времени накапливаются, прокрутка вниз; при переполнении старые строки отбрасываются) и **«Закрыть панель»**.
- Пока идёт выгрузка, повторные нажатия блокируются (`exportBusy`).

## 5. Источники кодов (по кнопкам)

1. **Текст в общем поле** — токены `[A-Za-z0-9_-]+`, порядок сохраняется, дубликаты убираются.
2. **Файл .txt** — содержимое выбранного файла, тот же разбор, что и для текста.
3. **TOURNAMENT-SHEDULE (CSV)** — кнопка слева; перед выбором файла вверху блока выставьте имена колонок (пресеты `TOURNAMENT_CODE` / `TOURNAMENT_STATUS` или свой ввод) и отметьте статусы в **левом** фильтре.
4. **TOURNAMENT-LIST (CSV)** — кнопка справа; те же сверху списки имён колонок; фильтр статусов — **правый**.

Разделитель строк CSV определяется по первой строке: если точек с запятой больше, чем запятых, используется `;`, иначе `,`. Поддерживается простое экранирование полей в кавычках.

## 6. Извлечение лидеров и запись в файл

Число лидеров берётся из `data.body.tournament.leaders`, иначе из `data.body.badge.leaders`. Пустое тело при **успешном** HTTP — ключ в JSON **не** добавляется. Ноль лидеров — в файл пишется **нормализованная** запись через `buildLeadersExportRecordArray` (см. §1). Для лога «участники» по сохранённым записям считаются вхождения непустого поля `employeeNumber` в дереве объекта.

## 7. Функции (кратко)

| Функция | Назначение |
|---------|------------|
| `parseTournamentCodesFromText` | Коды из произвольного текста |
| `parseCsv` | Разбор CSV в заголовки и строки |
| `codesFromCsvByColumns` | Отбор кодов по задаваемым именам колонок (код + статус) и фильтру чекбоксов |
| `fetchLeadersForAdmin` | Один GET |
| `countLeadersInResponseData` | Число лидеров в `data` или `null`, если ответ пустой |
| `buildLeadersExportRecordArray` | Какую запись положить в JSON по одному турниру (успех / 0 участников / ошибка / HTTP) |
| `countEmployeeNumberFieldsInTree` | Сколько раз в дереве ответа встречается непустое поле `employeeNumber` (для лога «участники») |
| `sanitizeExportFilenamePrefix` | Очистка строки префикса имени файла |
| `readRequestGapMs` (внутри панели) | Чтение паузы из поля ввода (мс) |
| `buildExportFilenamePrefix` (внутри панели) | Префикс имени до таймштампа (поле или авто) |
| `runExport` (внутри панели) | Цикл по кодам, сбор JSON, скачивание файла (если есть что сохранять) |
| `appendLabeledColumnSelect` (внутри панели) | `<select>` с пресетами имени колонки + «Другой…» и поле ввода |
| `createCsvSideBlock` (внутри панели) | Нижняя половина CSV: чекбоксы статусов своего формата, скрытый `input` файла, кнопка и `runExport` |
| `startTournamentPanel` | Сборка UI и обработчики кнопок |

Имя файла: `{префикс}{YYYYMMDD-HHMMSS}.json`, где **префикс** задаётся на панели или по умолчанию `leadersForAdmin_{STAND}_`.
