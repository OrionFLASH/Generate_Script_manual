# Скрипт выгрузки параметров по статусу (`Parameters_Actual_Export.js`)

## 1. Назначение

Скрипт запускается в DevTools и выполняет двухшаговую выгрузку параметров по выбранному статусу:

1. POST `.../proxy/v1/parameters` с телом `{ "status": "<ACTUAL|ARCHIVE>" }` — получить список;
2. из списка извлекаются `objectId`, и по каждому `objectId` выполняется отдельный POST `.../proxy/v1/parameters` с телом `{ "objectIds": ["<id>"] }`.

Итог сохраняется в JSON-файл.

## 2. Поддерживаемые окружения

На панели выбираются:

- Стенд: `PROM` / `PSI`;
- Контур: `ALPHA` / `SIGMA`.

Всего 4 варианта.

## 3. Запуск

- Открыть нужную страницу приложения и авторизоваться;
- DevTools → Console;
- Вставить код `Script/Parameters_Actual_Export.js`;
- Нажать Enter;
- На панели выбрать стенд/контур/статус и нажать `Запустить выгрузку`.

Скрипт обёрнут в IIFE, повторная вставка создаёт новый экземпляр панели без ошибки повторного `const`.

## 4. Панель

- Выбор стенда и контура;
- Выбор статуса для первого этапа: `ACTUAL` или `ARCHIVE`;
- Пауза между запросами детализации (`objectId`) в мс;
- Информационная строка с текущим endpoint;
- Кнопка `Запустить выгрузку`;
- Блок `Журнал работы` (подробные сообщения);
- Кнопка `Закрыть панель`.

В консоль выводятся только краткие сообщения о старте/завершении и ошибках верхнего уровня.

## 5. Формат выходного файла

Имя файла:

`parameters_<STAND>_<CONTOUR>_YYYY-MM-DD.json`

Пример:

`parameters_PROM_SIGMA_2026-04-15.json`

Структура:

- `meta` — служебные данные прогона (стенд, контур, endpoint, пауза, количество id, число успехов/ошибок);
- `list` — ответ первого запроса (`status: ACTUAL` или `status: ARCHIVE`);
- `details` — массив ответов по каждому `objectId`.

## 6. Основные константы и функции

| Имя | Назначение |
|-----|------------|
| `DEFAULT_STAND`, `DEFAULT_CONTOUR` | Значения по умолчанию (`PROM`, `SIGMA`) |
| `DEFAULT_STATUS` | Статус по умолчанию для этапа 1 (`ACTUAL`) |
| `PARAMETER_ORIGINS` | Таблица origin по стенду и контуру |
| `PARAMETERS_PATH` | Путь API (`/bo/rmkib.gamification/proxy/v1/parameters`) |
| `postParameters(origin, bodyObj)` | Универсальный POST к endpoint |
| `extractObjectIds(listData)` | Извлечение уникальных `objectId` из списка |
| `downloadJson(filename, obj)` | Сохранение результата в JSON |
| `startPanel()` | Построение панели и запуск сценария |

## 7. Важные замечания

- Куки берутся из текущей сессии браузера (`credentials: "include"`), в код не вшиваются.
- Пауза из панели применяется между запросами детализации (`objectId`), первый запрос выполняется сразу.
- При ошибках отдельных `objectId` выгрузка продолжается, информация фиксируется в `details` и `meta.detailErrors`.
- PSI-URL и PROM-URL задаются явно в `PARAMETER_ORIGINS`.
